# 儿童学习平台 - 全局UI设计规范
## 一、规范概述
本规范是全项目所有页面、功能模块必须遵循的统一设计标准，旨在保持产品体验一致性，降低开发沟通成本。所有新功能开发、现有功能迭代都必须严格遵循本规范，涉及全局更新时必须同步更新本规范文档。

## 三、导航规范
### 3.1 面包屑导航
所有二级及以下页面必须使用面包屑导航，统一规则：
1. **格式要求**：固定格式为 `首页 > 父级页面 > 当前页面`，支持无限层级扩展
2. **交互规则**：
   - 首页、父级页面支持点击跳转，使用灰色#666666字体
   - 父级页面hover时高亮显示主蓝色#3399FF
   - 当前页面使用对应模块的科目专属色，不可点击
3. **布局要求**：
   - 面包屑位于页面最顶部，距离内容区顶部24px
   - 字号统一为16px，行高24px
   - 分隔符使用中文大于号 `>`，左右间距各4px
4. **冗余禁止**：页面不得重复显示面包屑中已有的模块标题，避免信息重复
## 四、字体规范
1. **默认字体**：优先使用系统无衬线字体，顺序为：-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif
2. **字号层级**：
   | 字号 | 使用场景 |
   |------|----------|
   | 12px | 辅助说明、拼音标注 |
   | 14px | 正文内容、按钮文字 |
   | 16px | 面包屑、二级标题 |
   | 18-24px | 功能模块标题、诗词正文 |
   | 24-32px | 页面大标题 |
3. **行高规范**：正文行高1.5，标题行高1.2
## 五、通用组件规范
### 5.1 按钮规范
| 按钮类型 | 样式规范 | 使用场景 |
|---------|----------|----------|
| 主按钮 | 背景主蓝色#3399FF，白色文字，圆角8px，内边距12px 24px | 主要操作、提交按钮 |
| 次按钮 | 白色背景，灰色边框，黑色文字，圆角8px，内边距12px 24px | 次要操作、返回按钮 |
| 功能按钮 | 对应科目主题色背景，白色文字，圆角8px | 各模块内功能操作 |
- 按钮最小尺寸：≥48x48px，适合儿童点击
- 交互反馈：hover时轻微放大1.05倍，阴影加深；点击时有按压效果
### 5.2 卡片规范
- 背景白色，圆角12px，默认阴影0 2px 8px rgba(0,0,0,0.08)
- 内边距统一为24px
- hover时阴影加深为0 4px 16px rgba(0,0,0,0.12)，过渡动画0.2s
## 六、交互规范
1. **点击反馈**：所有可点击元素hover时必须有明显样式变化（颜色、背景、缩放等）
2. **操作反馈**：
   - 成功操作：绿色动画+星星奖励效果
   - 错误操作：红色提示+鼓励性文字
   - 加载状态：显示骨架屏或者加载动画，避免页面空白
3. **动画规范**：所有过渡动画统一使用0.2s ease缓动函数，避免过快或过慢的动画效果
## 七、文档更新规则
1. 本规范为全项目最高优先级设计标准，所有功能开发必须先遵循本规范
2. 涉及全局样式、组件、交互的更新，必须先更新本规范文档，再进行代码开发
3. 各科目模块的专属规范，在对应模块的需求文档中单独说明，但不得与本全局规范冲突
4. 规范迭代需要记录版本号和更新日期，历史版本可追溯
